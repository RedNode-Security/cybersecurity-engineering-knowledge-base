# Versioning

This repository uses lightweight semantic versioning for publication milestones.

## Version Types

| Version | Meaning |
|---|---|
| Patch | Corrections, typos, link fixes, small clarifications |
| Minor | New reference pages, rules, playbooks, or examples |
| Major | Structural change, new domain, or publication milestone |

## Status Labels

Repository version and page status are separate.

- Repository version describes release state.
- Page status describes content maturity.

A repository can have version `1.0.0-reference-candidate` while many pages remain
`draft`.

## Release Checklist

- [ ] Repository validation passes.
- [ ] Markdown lint passes.
- [ ] JSON samples validate.
- [ ] Published index is current.
- [ ] New pages have front matter.
- [ ] Unsafe content review completed.
- [ ] Release notes updated.

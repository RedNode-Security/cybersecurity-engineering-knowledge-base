# Start Here

## What this is

The Cybersecurity Engineering Handbook is a structured reference for defensive
security engineering. It connects concepts to practical work: what to log, what
to detect, how to triage, how to respond, and how to improve systems.

```text
Understand → Model → Detect → Respond → Automate → Improve
```

## Who it is for

- SOC analysts
- Detection engineers
- Incident responders
- Security engineers
- Cloud security engineers
- Application security engineers
- AI security practitioners
- Students and researchers building practical security knowledge

- [Published references](published-references.md)
- [Detection library](detection-library.md)
- [Site roadmap](site-roadmap.md)

## How to read it

Start with the published references. Then use the detection library and domain
folders as deeper reference material.

The repository uses content status labels:

| Status | Meaning |
|---|---|
| `draft` | Useful, but still being improved |
| `reviewed` | Checked for accuracy, clarity, and safety |
| `published` | Curated reference-grade page |
| `archived` | Kept for history, not actively maintained |
